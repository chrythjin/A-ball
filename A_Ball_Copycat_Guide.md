# 📜 엔트리 게임 기획 및 개발 설계도: "더 볼 (The Ball) - 카피캣"

본 문서는 엔트리의 대표적인 디펜스/로그라이크 명작 게임인 **'A Ball'**의 핵심 재미 요소를 분석하고, 이를 엔트리(Entry) 플랫폼에서 직접 구현할 수 있도록 작성된 상세 개발 가이드(프롬프트)입니다.

---

## 🎮 1. 게임 기본 정보

* **게임 제목 (가제):** 더 볼 (The Ball)
* **장르:** 로그라이크 디펜스 (벽돌깨기 + 타워 디펜스)
* **핵심 콘셉트:** 단 하나의 공을 발사해, 매 턴마다 위에서 아래로 내려오는 적 블록들을 파괴하여 플레이어의 라인을 사수하는 게임입니다.
* **플랫폼:** 엔트리 (Entry)

---

## 🖼️ 2. 장면 (Scene) 및 오브젝트 구성

엔트리 프로젝트에서 생성해야 할 장면과 각 장면에 필요한 오브젝트 목록입니다.

### 🎬 장면 1: 로비 (Lobby)
* **역할:** 게임 진입 화면 및 시작 대기 공간
* **필요 오브젝트:**
    * `게임 제목`: 글상자 또는 폰트 이미지 ("THE BALL")
    * `시작 버튼`: 클릭 시 '장면 2(인게임)'로 전환되는 버튼

### 🕹️ 장면 2: 인게임 (In-game)
* **역할:** 실제 게임 플레이가 이루어지는 핵심 화면
* **배경:** 컴컴한 단색 배경 (벽돌과 공의 궤적이 잘 보이도록 어두운 톤 권장)
* **필요 오브젝트:**
    1.  `공 발사대`: 화면 중앙 하단에 고정된 오브젝트 (마우스 방향을 조준하는 기준점)
    2.  `공 (플레이어)`: 작은 원형 오브젝트 (물리 이동 및 충돌의 주체)
    3.  `적 (벽돌)`: 사각형 또는 육각형 오브젝트 (체력을 가지고 아래로 내려옴)
    4.  `데드라인 (바닥 선)`: 화면 최하단에 위치한 빨간색 선 (적이 닿으면 게임 오버)
    5.  `스킬 선택 창`: 평소에는 숨겨져 있다가 특정 턴에 나타나는 팝업 UI
    6.  `스킬 버튼 (1, 2, 3)`: 팝업창 위에 배치되는 3개의 선택지 버튼
    7.  `UI 글상자`: 현재 턴, 공의 개수, 공격력 등을 실시간으로 표시하는 텍스트

## 🛠️ 3. 속성 설정 (변수 및 리스트)

코딩을 시작하기 전, 엔트리의 **[속성]** 탭에서 반드시 생성해야 하는 요소들입니다.

### ① 변수 (Variables)
| 변수명 | 기본값 | 설명 |
| :--- | :---: | :--- |
| `현재_턴` | 1 | 게임이 진행 중인 현재 라운드/턴 수 |
| `게임_상태` | 0 | 0: 조준 대기, 1: 공 발사 중, 2: 게임 오버 |
| `공_데미지` | 1 | 공이 적에게 주는 타격치 |
| `공_개수` | 1 | 한 턴에 발사되는 총 공의 개수 |
| `공_속도` | 15 | 공이 이동하는 속도 |
| `현재_날아간_공` | 0 | 현재 화면에 발사되어 움직이고 있는 공의 개수 |
| `적_최대_체력` | 1 | 새로 생성되는 적의 기준 체력 (턴이 지날수록 증가) |
| `스킬_선택중` | 거짓 | 스킬 팝업창이 떠서 게임이 일시 정지되었는지 여부 |

### ② 리스트 (Lists)
* **`적_체력_리스트`**: 화면에 존재하는 모든 적들의 체력을 개별적으로 관리하기 위한 리스트입니다.

## 💻 4. 오브젝트별 핵심 로직 및 코드 구조

### 🟢 A. 공 발사대 & 공 (Ball) 시스템
공의 발사각을 조준하고, 턴에 맞춰 공을 순차적으로 사출하는 로직입니다.

1.  **마우스 조준 (발사대 뱡향 제어):**
    * `[무한 반복하기]` -> `[만약 <게임_상태 = 0> 이라면]`
    * `[마우스 포인터 쪽 바라보기]` 블록을 사용해 발사대의 각도를 조준선처럼 움직입니다.
2.  **공 발사 메커니즘:**
    * `[마우스를 클릭했을 때]` -> `[만약 <게임_상태 = 0> 이라면]`
    * `게임_상태`를 `1(발사 중)`로 변경합니다.
    * `[공_개수 번 반복하기]` 루프를 돌리며 `[공 오브젝트의 자신의 복제본 만들기]`를 실행합니다. (사출 간격을 위해 사이에 `0.1초 기다리기` 삽입)
3.  **공 복제본의 이동 및 충돌 (벽돌깨기 핵심):**
    * `[복제본이 생겼을 때]` -> `현재_날아간_공`을 `1`만큼 증가시킵니다.
    * 발사대의 이동 방향을 복제본의 이동 방향으로 설정합니다.
    * `[무한 반복하기]` 구조 안에서 `[이동 방향으로 (공_속도) 만큼 움직이기]`를 수행합니다.
    * `[만약 <벽에 닿았는가> 이라면]` -> `[벽에 닿으면 튕기기]` 블록 사용.
    * `[만약 <적(벽돌)에 닿았는가> 이라면]` -> 적에게 데미지를 전달하고, 튕기는 각도를 계산하여 이동 방향을 전환합니다.
4.  **수거 및 턴 종료 조건:**
    * 공 복제본이 화면 바닥(데드라인)에 닿으면 `현재_날아간_공`을 `-1` 하고 `[이 복제본 삭제하기]`를 실행합니다.
    * 발사대 오브젝트에서는 `[무한 반복하기]`로 감시하다가 `[만약 <현재_날아간_공 = 0> 이고 <게임_상태 = 1> 이라면]` 조건이 충족되면 **"턴 종료"** 신호를 보냅니다.

### 🟥 B. 적 (Enemy) 시스템
턴제 디펜스 방식으로 적이 생성되고 한 칸씩 전진하는 핵심 시스템입니다.

1.  **적 생성 및 갱신 ("턴 종료" 신호를 받았을 때):**
    * 현재 턴을 1 증가시킵니다.
    * `적_최대_체력` 변수를 난이도 공식(예: `현재_턴 * 1.2`)에 맞춰 상승시킵니다.
    * 화면 상단(Y: 120 부근)의 X 좌표를 무작위로 설정하여 적 복제본을 수 개 생성합니다.
    * 생성된 적의 체력을 `적_체력_리스트`에 기록합니다.
2.  **적의 전진 메커니즘:**
    * "턴 종료" 신호가 올 때마다, 기존에 살아있는 모든 적 복제본은 `[Y 좌표를 -40만큼 바꾸기]`를 실행하여 아래로 내려옵니다.
3.  **피격 및 소멸:**
    * 공 복제본과 닿으면 자신의 체력을 `공_데미지`만큼 감소시킵니다.
    * 체력이 0 이하가 되면 `적_체력_리스트`에서 자신의 항목을 제거하고 `[이 복제본 삭제하기]`를 수행합니다.
4.  **게임 오버 (Defeat) 판정:**
    * 적 복제본의 Y 좌표를 상시 체크하여 `[만약 <Y 좌표 < -140 (데드라인)> 이라면]` -> `게임_상태`를 `2`로 바꾸고 **'장면 3 (게임오버)'**으로 이동합니다.

### 🌟 C. 로그라이크 스킬 (Upgrade) 시스템
게임의 중독성을 배가시키는 랜덤 강화 선택 시스템입니다.

1.  **스킬 턴 감지:**
    * "턴 종료" 신호 처리 도중 `[만약 <(현재_턴) / 10 의 나머지 = 5> 이라면]` (즉 5턴, 15턴, 25턴...) 조건문을 실행합니다.
2.  **게임 일시정지 및 팝업:**
    * `스킬_선택중` 변수를 `참`으로 설정하여 공 발사 및 적 이동 로직을 잠시 멈춥니다.
    * `스킬 선택 창` 오브젝트에 "보이기" 신호를 보냅니다.
3.  **스킬 버튼 기능 구현 (예시):**
    * **1번 버튼 (파워 업):** 클릭 시 `공_데미지` 변수를 `+1` 시키고 팝업창을 닫음.
    * **2번 버튼 (멀티 볼):** 클릭 시 `공_개수` 변수를 `+2` 시키고 팝업창을 닫음.
    * **3번 버튼 (스피드 업):** 클릭 시 `공_속도` 변수를 `+3` 시키고 팝업창을 닫음.
4.  **복귀:**
    * 선택이 끝나면 팝업창을 숨기고 `스킬_선택중`을 `거짓`, `게임_상태`를 `0(조준 대기)`으로 돌려 다음 게임 플레이를 재개합니다.

## 🚀 5. 단계별 개발 권장 순서 (로드맵)

* **1단계: 프로토타입 (MVP) 만들기**
    * 스킬이나 리스트 구현은 미뤄두고, **공 1개 발사 -> 벽에 튕기기 -> 바닥에 닿으면 소멸 -> 턴 종료**로 이어지는 기본 물리 루프만 먼저 완벽하게 구현하세요.
* **2단계: 적 배치 및 전진 로직 연동**
    * 적이 상단에 스폰되고, 공이 다 돌아오면 한 칸 아래로 내려오는 턴 메커니즘을 결합합니다.
* **3단계: 복제본 및 데이터 연동**
    * 공의 개수를 늘려 여러 개가 발사되게 만들고, 적의 체력을 리스트로 관리하여 다단 히트 분쇄 효과를 구현합니다.
* **4단계: 로그라이크 업그레이드 요소 도입**
    * 5턴 주기로 스킬 창이 뜨고 변수가 강화되는 재미 요소를 붙입니다.
* **5단계: 폴리싱 (Polishing)**
    * 화려한 타격 이펙트, 화면 흔들림 효과, 튕길 때와 터질 때의 사운드를 추가하여 완성도를 극대화합니다.
