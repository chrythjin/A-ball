# 🖼️ 더 볼 (The Ball) - UI/UX 디자인 및 레이아웃 명세서

본 문서는 **'더 볼 (The Ball) - 카피캣'** 게임의 UI 구성, 화면 레이아웃, 스타일링 요소, 그리고 오브젝트 사양을 빠짐없이 정리한 디자인 명세서입니다. 기획서(`A_Ball_Copycat_Guide.md`)와 실제 구현된 HTML/CSS/JS 코드를 기반으로 일치되도록 작성되었습니다.

---

## 🎨 1. 기본 스타일 및 공통 사양

### ① 화면 규격 (Game Container)
* **크기:** 가로 480px × 세로 640px (`#game-container`)
* **위치:** 화면 중앙 정렬 (`display: flex`, `justify-content: center`, `align-items: center`)
* **배경색:**
  * 전체 브라우저 배경: `#0f0f1a` (어두운 사이버 펑크 스타일)
  * 게임 판(Container/Canvas) 배경: `#1a1a2e` (어두운 남색 톤)
* **기타:** 외부 영역 흘러넘침 방지 (`overflow: hidden`), 부드러운 그림자 효과 적용 (`box-shadow: 0 0 20px rgba(0,0,0,0.5)`)

### ② 공통 타이포그래피 & 색상
* **기본 폰트:** `sans-serif` (추후 Orbitron, Inter 등 모던 웹 폰트 적용 권장)
* **텍스트 색상:** 기본 흰색 (`white`) 및 연회색 테두리
* **액센트 색상:**
  * 기본 보라/남색: `#4a4e69` (버튼 기본 배경색, 테두리 등)
  * 선택/호버 색상: `#9a8c98` (버튼 호버 시)
  * 빨간색 테두리/에너지선: `#ff4d4d` (데드라인 및 경고 요소)
  * 배경 반투명색: `rgba(26, 26, 46, 0.95)` (팝업창 배경)

## 🎬 2. 장면(Scene)별 상세 UI 사양

### 1️⃣ 장면 1: 로비 (Lobby Scene)
* **ID:** `#scene-lobby` (활성화 시 `.active` 클래스로 `display: flex` 적용)
* **요소 구성:**
  1. **게임 제목 (`#game-title`):** 
     * **텍스트:** `THE BALL`
     * **스타일:** 글자 크기 48px, 자간(letter-spacing) 2px, 굵게(bold) 표시, 아래 마진 40px
  2. **시작 버튼 (`#start-btn`):**
     * **텍스트:** `시작`
     * **스타일:** 안쪽 여백 `15px 40px`, 글자 크기 24px, 둥근 모서리 `8px`, 테두리 없음, 배경색 `#4a4e69`
     * **이펙트:** 마우스 호버 시 배경색이 `#9a8c98`로 부드럽게 전환 (`transition: background-color 0.2s`)
     * **동작:** 클릭 시 `startGame()`이 호출되어 인게임 장면으로 전환

---

### 2️⃣ 장면 2: 인게임 (In-game Scene)
* **ID:** `#scene-ingame` (활성화 시 `.active` 클래스 적용, 마우스 클릭을 캔버스로 전달하기 위해 `pointer-events: none` 설정)
* **요소 구성:**

#### A. 상단 HUD (Heads-Up Display - `#ui-hud`)
* **스타일:** 가로 100% 영역을 차지하는 플렉스바 (`display: flex`, `justify-content: space-between`), 안쪽 여백 `15px`, 글자 크기 18px, 굵게(bold), 클릭 차단 해제 (`pointer-events: auto`)
* **실시간 표시 정보 (hud.js 연동):**
  * **턴 (Turn):** `gameState.현재_턴`
  * **공 (Balls):** `gameState.공_개수`
  * **데미지 (Damage):** `gameState.공_데미지`
  * **속도 (Speed):** `gameState.공_속도`
  * **상태 (Status):** `gameState.게임_상태`
  * **날아간 공:** `gameState.현재_날아간_공`
  * *형식:* `턴: X | 공: Y | 데미지: Z | 속도: S | 상태: A | 날아간공: B`

#### B. 캔버스 영역 (Canvas - `#gameCanvas`)
* **크기:** 가로 480px × 세로 640px
* **그려지는 그래픽 요소:**
  1. **공 발사대 (Launcher):**
     * **위치:** 하단 중앙 (X: 220, Y: 560), 크기 가로 40px × 세로 40px 직사각형 (`#4a4e69`)
  2. **대기 중인 공 (Ball Starter):**
     * **위치:** (X: 240, Y: 545), 반지름 8px 원형 (`#f2e9e4`)
  3. **조준선 (Aim Line):**
     * **시작점:** (X: 240, Y: 560)에서 마우스 위치 방향으로 뻗어나감
     * **최대 길이:** 200px 제한 (`getLaunchAngle`로 라디안 각 계산)
     * **스타일:** 점선 형태 (`setLineDash([5, 5])`), 두께 2px, 선 색상 `#f2e9e4`
  4. **날아가는 공 (Ball Clones):**
     * **크기:** 반지름 8px 원형
     * **색상:** `#f2e9e4`
  5. **적 블록 (Enemy Bricks):**
     * **크기:** 가로 60px × 세로 30px 직사각형
     * **배치 영역:** 좌측 여백 40px ~ 우측 여백 440px 범위 내에서 균등 분할 생성
     * **체력 비례 렌더링:** 체력 비율(현재 체력 / 최대 체력)에 따라 **초록색(Green)에서 빨간색(Red)**으로 변하는 동적 RGB 컬러 채우기 (`rgb(r, g, 0)`)
     * **체력 텍스트:** 블록 중앙에 흰색 볼드체 14px 폰트로 남은 HP 숫자 텍스트 표시
  6. **데드라인 (Deadline):**
     * **위치:** Y: 600, 두께 5px 선 (`#ff4d4d`)
     * **실제 게임오버 트리거 조건:** 적의 바닥 모서리(Y + 30px)가 **580px**에 도달하거나 넘어서면 즉시 게임 오버 (`checkGameOver`)

#### C. 디버그 리스트 (Debug Info - `#debug-list`)
* **역할:** 화면 상에 존재하는 적들의 체력 정보가 동기화된 리스트 상태를 모니터링하기 위한 임시 HUD 텍스트 영역

#### D. 스킬 선택 팝업창 (Skill Selection Popup - `#skill-popup`)
* **스타일:**
  * 화면 정중앙 배치 (`position: absolute`, `top: 50%`, `left: 50%`, `transform: translate(-50%, -50%)`)
  * 반투명 진남색 배경 (`rgba(26, 26, 46, 0.95)`), 둥근 모서리 `12px`, 테두리 `2px solid #4a4e69`, 입체적인 바깥 그림자 (`box-shadow: 0 10px 30px rgba(0,0,0,0.8)`)
  * 클릭 활성화 (`pointer-events: auto`), Z축 최상위 배치 (`z-index: 10`)
* **내부 구성:**
  * **팝업 제목:** `스킬 선택` (마진 바텀 20px, 연분홍 톤 `#f2e9e4` 색상)
  * **버튼 3종:**
    1. **1번 버튼 (`#skill-btn-1`):** `파워 업 (데미지+1)`
    2. **2번 버튼 (`#skill-btn-2`):** `멀티볼 (공+2)`
    3. **3번 버튼 (`#skill-btn-3`):** `스피드 업 (속도+3)`
    * *버튼 개별 사양:* 가로 폭 100%, 상하 마진 10px, 패딩 `12px 20px`, 글자 크기 18px

#### E. 게임오버 테스트 버튼 (`#gameover-trigger`)
* **역할:** 개발 및 검증용 임시 게임 오버 트리거 버튼
* **스타일:** 최하단 위치, 반투명 빨강 배경 (`rgba(255, 77, 77, 0.2)`), 테두리 `#ff4d4d`, 패딩 `10px 20px`

### 3️⃣ 장면 3: 게임 오버 (Game Over Scene)
* **ID:** `#scene-gameover` (활성화 시 `.active` 클래스로 `display: flex` 적용)
* **요소 구성:**
  1. **게임오버 타이틀:**
     * **텍스트:** `게임 오버`
     * **스타일:** 글자 크기 48px, 아래 마진 40px, 빨간색/경고 스타일 강조
  2. **재시작 버튼:**
     * **텍스트:** `다시 시작`
     * **스타일:** 1번 장면의 시작 버튼과 동일 스타일링
     * **동작:** 클릭 시 로비 화면 (`scene-lobby`)으로 이동하고, 게임 데이터를 초기 상태로 완전히 초기화 (`resetGame()`)

## ⚡ 3. UI/UX 인터랙션 시나리오 및 예외 처리

1. **상태 게이팅 (State Gating):**
   * 공이 날아다니는 동안(`게임_상태 === 1`) 또는 스킬 팝업창이 떠 있는 동안(`스킬_선택중 === true`)에는 캔버스 마우스 클릭이 감지되어도 추가 발사가 차단(무시)됩니다.
2. **조준선 제한:**
   * 조준 각도가 발사 지점보다 위쪽 영역을 향하도록 하며, 최대 200px 길이로 제한하여 조준의 정확도를 높이고 너무 긴 점선이 화면을 어지럽히지 않도록 제한합니다.
3. **스킬 선택 중 대기 상태:**
   * 팝업이 뜨면 `스킬_선택중` 변수가 `true`로 갱신되어 루프 단계의 상태 업데이트 및 적의 추가 전진 등이 멈춥니다. 스킬 선택이 끝나 팝업이 닫히면 다시 `스킬_선택중 = false`, `게임_상태 = 0`으로 돌아가며 플레이가 진행됩니다.
